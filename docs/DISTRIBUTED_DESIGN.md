## 愿景

> 把家里闲置设备组成 AI 集群，零配置，即插即用。

Hippo 从"Ollama 的 Python 替代品"进化为**轻量级分布式 LLM 部署平台**。

## 差异化定位

| 竞品 | 定位 | Hippo 的不同 |
|------|------|-------------|
| Ollama | 单机本地推理 | Hippo 支持多设备分布式 |
| vLLM | 企业级分布式 | Hippo 轻量，家庭设备可用 |
| Exo | P2P 分布式 | Hippo 有路由/治理/Reranker |

**独特组合**：分布式 + 意图路由 + LFRU 缓存 + 治理层 + Reranker

## 架构

```
                    ┌─────────────────┐
                    │   Hippo Gateway  │ ← 统一 API（兼容 OpenAI）
                    │   (主节点)        │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
     ┌────────┴──┐   ┌──────┴─────┐   ┌────┴──────┐
     │ Worker A  │   │ Worker B   │   │ Worker C  │
     │ 5060 Ti   │   │ Y9000P     │   │ Mac Mini  │
     │ 16GB VRAM │   │ 8GB VRAM   │   │ 16GB 统一  │
     └───────────┘   └────────────┘   └───────────┘
```

### 角色

- **Gateway（网关）**：统一入口，路由调度，模型分片管理
- **Worker（工作节点）**：注册 GPU 资源，执行推理，报告状态

### 关键组件

| 组件 | 职责 |
|------|------|
| `cluster/discovery.py` | mDNS 自动发现局域网 Hippo 实例 |
| `cluster/worker.py` | Worker 注册 + 心跳 + 资源报告 |
| `cluster/scheduler.py` | 按显存/算力自动分片模型层 |
| `cluster/transport.py` | 隐藏状态跨设备传输（TCP） |
| `cluster/gateway.py` | 统一 API 网关，对外一个端口 |

### 分片策略

```
Qwen3.6-35B-A3B（22GB）在 16GB + 8GB 集群上：

Worker A (16GB): Layer 0-31 (14GB)
Worker B (8GB):  Layer 32-47 (6GB)
剩余 2GB: KV cache
```

自动分配算法：
1. 收集所有 Worker 的显存和算力
2. 按显存比例分配层数
3. 算力强的 Worker 分配更多连续层（减少通信）
4. 考虑网络延迟（同一交换机优先配对）

### 容错

| 场景 | 处理 |
|------|------|
| Worker 掉线 | 心跳超时 30s → 标记离线 → 暂停推理 → 尝试单机降级 |
| Worker 重新上线 | 自动重新注册 → 重新分片 → 恢复服务 |
| 新 Worker 加入 | 触发重新分片（可选：等待当前请求完成） |

## 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| 设备发现 | mDNS (zeroconf) | 零配置，局域网自动发现 |
| 通信协议 | TCP + MessagePack | 比 gRPC 轻，比 REST 快 |
| 分片格式 | GGUF（沿袭现有） | llama.cpp 生态兼容 |
| API 兼容 | OpenAI API 格式 | 兼容所有现有客户端 |

## 实施路线

### Phase 1：基础集群（1-2 周）

**目标**：两台机器跑一个模型

- [ ] `hippo worker start` — 启动 Worker 模式
- [ ] `hippo gateway start` — 启动 Gateway 模式
- [ ] mDNS 自动发现
- [ ] 手动分片配置（routes.json 扩展）
- [ ] 基础 API 透传
